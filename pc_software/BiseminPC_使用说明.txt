Bisemin PC Control 软件使用说明
================================

一、软件用途
------------
Bisemin PC Control 用于通过串口连接 MCU，对两个温控池进行监控和控制。
软件支持普通控温、程序控温、状态查看、数据记录和 CSV 导出。


二、运行环境
------------
1. MCU 已上电并正常运行。
2. PC 与 MCU 已通过串口连接。
3. 推荐使用默认波特率 1000000。
4. 如果使用源码运行，需要已安装 Python 和 requirements.txt 中的依赖。
5. 如果使用 exe 运行，直接打开 dist\BiseminPC.exe 即可。


三、启动软件
------------
方式 1：源码运行
在 PowerShell 中执行：

    cd D:\bisemin\bisemin\pc_software
    python main.py

如果提示缺少依赖，先执行：

    pip install -r requirements.txt

方式 2：exe 运行
直接运行：

    D:\bisemin\bisemin\pc_software\dist\BiseminPC.exe


四、串口连接
------------
1. 打开软件后，在 Port 下拉框中选择对应串口。
2. Baud 选择波特率，默认建议使用 1000000。
3. 点击 Connect 连接设备。
4. 连接成功后，按钮会变为 Disconnect。
5. 可以点击 HELLO 检查 MCU 是否有响应。
6. 可以点击 GET_STATE 主动读取当前状态。

如果没有看到串口：
1. 确认 USB/串口线已连接。
2. 确认驱动已安装。
3. 点击 Refresh 刷新串口列表。
4. 确认串口没有被其他软件占用。


五、主界面说明
--------------
软件左侧为两个温控池控制面板：

    Cell 0 对应温控池 1
    Cell 1 对应温控池 2

每个温控池都有独立的状态显示、普通控温和程序控温操作区域。

右侧为日志窗口：

    Data Log：数据记录表格
    RX：接收到的串口帧
    TX：发送出去的串口帧


六、温控池状态说明
------------------
Mode：当前运行模式。

    STOP    停止
    NORMAL  普通控温
    PROGRAM 程序控温

Owner：当前控制来源。

    NONE  无控制来源
    PANEL 面板控制
    PC    PC 软件控制

Running：是否正在运行。

    0 未运行
    1 正在运行

Current：当前温度，单位摄氏度，显示到小数点后一位。

Target：当前目标温度，单位摄氏度，显示到小数点后一位。

Duty：当前输出占空比。

Error：错误码。0 表示无错误，非 0 表示存在错误，需要结合 MCU 错误定义判断具体原因。


七、普通控温使用方法
--------------------
普通控温用于让某一个温控池保持在指定目标温度。

操作步骤：
1. 在对应温控池的 Normal 区域设置 Target。
2. 点击 Start Normal。
3. 观察 Current 是否逐渐接近 Target。
4. 如需停止，点击 Stop。

说明：
1. Stop 只停止当前温控池，不会影响另一个温控池。
2. 普通控温和程序控温不要对同一个温控池同时操作。
3. 如果温控池已经处于程序控温，建议先 Stop Program 或 Stop 后再启动普通控温。


八、程序控温参数说明
--------------------
程序控温用于按照设定流程改变目标温度。

Program 区域参数如下：

Start：起始温度，单位摄氏度。
MCU 启动程序控温后，会先让命令温度到达 Start。

Hold：起始温度保持时间，单位秒。
到达 Start 后，保持 Hold 秒。

Rate：升降温速率，单位摄氏度/分钟。
从当前阶段温度变化到 Next 时使用该速率。

Next：下一目标温度，单位摄氏度。
完成起始保持后，温度会按照 Rate 变化到 Next。

Wait：到达 Next 后等待时间，单位秒。
每次到达一个 Next 阶段目标后，会等待 Wait 秒。

Repeat：重复推进次数。
MCU 会在到达 Next 并等待后，根据 Repeat 决定后续推进次数。
Repeat 越大，程序控温过程持续越久。


九、程序控温使用方法
--------------------
操作步骤：
1. 在对应温控池的 Program 区域设置 Start、Hold、Rate、Next、Wait、Repeat。
2. 点击 Set Program，将参数下发到 MCU。
3. 点击 Start Program，启动程序控温。
4. 观察 Mode 是否变为 PROGRAM。
5. 如需停止，点击 Stop Program。

注意：
1. 修改参数后，必须先点击 Set Program，再点击 Start Program。
2. Start Program 只负责启动已经下发到 MCU 的程序参数。
3. Stop Program 复用 MCU 的 STOP 命令，只停止当前温控池。
4. 程序控温运行中一般不建议重新 Set Program。如需修改，建议先停止再重新设置。


十、程序控温 phase 说明
-----------------------
phase 是 MCU 内部程序控温的阶段编号，用于说明当前程序执行到哪一步。

phase 含义如下：

    0 停止，或未处于程序控温
    1 正在到达起始温度 Start
    2 起始温度保持 Hold
    3 按 Rate 到达下一目标温度 Next
    4 到达 Next 后等待 Wait
    5 程序完成

为什么需要 phase：
程序控温不是一个单一目标温度，而是一个过程。MCU 需要知道当前处于到达起始温度、保持、升降温、等待还是完成阶段。记录 phase 后，后续分析数据时可以判断温度变化属于哪个程序阶段。


十一、数据记录与导出
--------------------
Data Log 页面用于记录 MCU 上报的数据。

Start Log：开始记录数据。
Stop Log：停止记录数据。
Export CSV：将当前记录导出为 CSV 文件。

当前记录字段：

mcu_time_ms：MCU 时间，单位 ms。
pool：温控池编号。

    Pool 1 对应温控池 1
    Pool 2 对应温控池 2

mode：运行模式，STOP、NORMAL 或 PROGRAM。
phase：程序控温阶段编号。
target：目标温度，单位摄氏度，保留一位小数。
current：当前温度，单位摄氏度，保留一位小数。
duty：输出占空比。
error：错误码。

说明：
1. 只有点击 Start Log 后，软件才会记录 DATA 数据。
2. 点击 Stop Log 后停止记录，但已记录的数据仍保留在表格中。
3. 点击 Export CSV 可以导出当前表格中的记录。
4. 重新点击 Start Log 会清空旧表格并开始新的记录。
5. CSV 文件使用 UTF-8 BOM 编码，方便 Excel 打开。


十二、RX/TX 调试窗口
--------------------
RX 页面显示 PC 收到的串口帧。
TX 页面显示 PC 发出的串口帧。

这些页面主要用于调试通信问题。
普通使用时主要关注左侧温控池状态和 Data Log 即可。


十三、常见问题
--------------
1. 点击 Connect 后没有数据

检查串口号是否正确，波特率是否为 1000000，MCU 是否已上电，串口是否被其他软件占用。

2. 点击 Start Program 后没有按预期运行

确认已经先点击 Set Program。Start Program 启动的是 MCU 中已经保存的程序参数。

3. 程序控温如何停止

点击对应温控池 Program 区域的 Stop Program。也可以点击 Normal 区域的 Stop。两者都会停止当前温控池。

4. 为什么有 Cell 0 和 Cell 1

这是软件内部和 MCU 协议使用的编号方式。为了方便操作者理解，数据记录中会显示为 Pool 1 和 Pool 2。

5. 为什么温度只显示一位小数

当前软件界面和数据记录表格将温度显示到小数点后一位，便于观察和记录。MCU 协议中也按一位小数发送主要温度字段。


十四、注意事项
--------------
1. 不要同时使用多个上位机软件连接同一个串口。
2. 对同一个温控池，不要同时进行普通控温和程序控温。
3. 修改程序控温参数后，需要重新 Set Program。
4. 停止操作只影响当前温控池，不影响另一个温控池。
5. 如果出现错误码，建议先停止控温，检查硬件状态后再重新启动。
6. 导出数据前请确认已经点击 Start Log 并记录到了有效数据。
